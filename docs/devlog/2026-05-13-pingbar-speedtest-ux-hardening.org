* [2026-05-13] Dev Log: PingBar speed-test and UX hardening       :RELIABILITY:UX:OPS:

** Context
PingBar is a native macOS menu bar network monitor in =/Users/eric/wrk/speedtest/worktree/pingbar=.

** Why
The in-app Cloudflare speed test failed before measurement because =https://speed.cloudflare.com/meta= returns =403= without a =Referer: https://speed.cloudflare.com/= header, and the Swift decoder expected =colo= to be a string while Cloudflare returns a nested object.

** What
- Fixed =SpeedTestRunner= metadata requests by matching the working Go CLI protocol: send Cloudflare =Referer=, use =GET /__down?bytes=0&measId=...= for latency, attach =measId= to download/upload probes, and validate HTTP status codes.
- Changed speed-test execution from all-or-nothing to partial-result reporting so metadata, latency, download, or upload phase failures do not hide successful measurements.
- Made =Direct= speed tests and direct public-IP checks disable HTTP, HTTPS, and SOCKS proxy settings through =URLSessionConfiguration.connectionProxyDictionary=.
- Added stable menu bar width settings to reduce status-item flicker while live throughput text changes.
- Added Wi-Fi/Network System Settings entry points and exposed active interface/gateway in the popover and copied diagnostic report.
- Loaded speed-test history during =NetworkState= initialization so pinned/floating windows have history without requiring a popover =onAppear= pass.

** How
*** Steps
1. Read prior dev logs in =docs/devlog/= and current app files under =PingBar/=.
2. Reproduced Cloudflare metadata failure with =curl=.
3. Compared Swift runner behavior to =worktree/speedtestcli/pkg/speedtest/cfbackend/meta.go= and =measure.go=.
4. Patched Swift runner, settings, UI sections, and system-settings utility.

  - =Direct= is now =No URL Proxy=.
  - =Proxy= is now =System Proxy= only when macOS HTTP/HTTPS/SOCKS proxy settings exist.
  - Otherwise the normal URLSession path is labeled =System Path=.
- Disabled PAC and proxy auto-discovery for no-url-proxy sessions in both public-IP probes and speed tests.

- Added =scripts/generate_app_icon.py= to regenerate icon sizes from =PingBar/Resources/Brand/PingBarAppIconSource.png= without ImageMagick/Pillow.

*** Route semantics
No URL Proxy means URLSession was configured not to use HTTP, HTTPS, SOCKS, PAC, or proxy auto-discovery. It does not and cannot bypass a VPN, WARP, or TUN interface because those sit below URLSession routing.

System Path means URLSession uses macOS system network settings. If a system URL proxy is configured, this should reveal the proxy egress. If only a VPN/tunnel is active, No URL Proxy and System Path can legitimately report the same VPN egress IP.
